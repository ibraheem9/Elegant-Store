# Elegant Store - نظام إدارة متجر البيع بالتجزئة 🛍️

نظام متكامل واحترافي لإدارة عمليات البيع، المشتريات، والديون، مصمم خصيصاً ليعمل بكفاءة عالية **بدون إنترنت (Offline-first)** مع واجهة مستخدم عصرية تدعم اللغة العربية بالكامل وتناسب أجهزة الكمبيوتر والأجهزة المحمولة.

---

## 🚀 الرؤية التقنية (Technical Vision)
تم بناء النظام باستخدام **Flutter** و **SQLite** لضمان استجابة فورية وحفظ دائم للبيانات محلياً. يتبع النظام معايير هندسة البرمجيات الحديثة (Clean Code & SOLID) لضمان سهولة الصيانة والتوسع المستقبلي.

## 🌟 المميزات الرئيسية (Key Features)

### 1. شاشة البيع الذكية (Smart Sales Screen)
- **بحث متطور (Autocomplete)**: نظام ذكي للبحث عن الزبائن يظهر فوق العناصر الأخرى لضمان سهولة الاستخدام.
- **إدارة المديونية والحدود**: النظام يمنع تلقائياً تجاوز سقف الدين للزبائن الدائمين ويصدر تنبيهات عند الوصول لـ 90% من الحد.
- **منطق الرصيد الموحد**: تتبع دقيق للرصيد (الموجب رصيد مودع، السالب دين قائمة) مع تسجيل كل حركة في سجل المعاملات (Transactions).
- **إيداع الرصيد (Prepayment)**: 🆕 نظام "دفع مقدم" يتطلب اختيار وسيلة الإيداع (كاش أو تطبيق) بشكل إلزامي لضمان دقة التقارير.
- **السحب النقدي (Cash Withdrawal)**: إمكانية تسجيل مبالغ مسحوبة نقداً من الصندوق كدين على المشتري، مع ضمان توازن الجرد اليومي.

### 2. واجهة المستخدم والترميز اللوني (Visual Coding)
تعتمد البطاقات نظام أشرطة ملونة في الأسفل لتمييز الحالة المالية بسرعة:
- 🟢 **الشريط الأخضر**: فواتير "دفع مقدم (إيداع رصيد)".
- 🔵 **الشريط الأزرق**: الفواتير المدفوعة بالكامل (SALE).
- 🟠 **الشريط البرتقالي**: فواتير الدين (قائمة/جزئية) وعمليات السحب النقدي.

### 3. المحاسبة والجرد اليومي (End of Day Statistics)
تطبيق معادلة محاسبية دقيقة لجرد الصندوق وحساب الأرباح:
> **دخل اليوم الصافي** = (صندوق اليوم + مشتريات الكاش) - (صندوق الأمس + سداد الديون كاش)

### 4. الإعدادات وإدارة البيانات (Settings & Backup)
- **تصدير البيانات**: إمكانية تصدير كافة بيانات المتجر كملف JSON للاحتفاظ بنسخة احتياطية.
- **تفريغ البيانات**: 🆕 خيار متقدم لحذف كافة الفواتير والعمليات المالية مع تصفير الأرصدة والحفاظ على بيانات الزبائن وطرق الدفع.
- **الوضع الليلي**: دعم كامل للمظهر الداكن (Dark Mode) لراحة العين.

---

## 📖 سيناريو العمل اليومي (Daily Sales Workflow)

1.  **بدء العملية**: يدخل المحاسب اسم المشتري (النظام يقترح الأسماء فوراً).
2.  **تحديد المبلغ**: إدخل المبلغ وتحديد طريقة الدفع (كاش، تطبيق، من الرصيد).
3.  **إيداع رصيد**: عند استلام مبلغ كمقدم، يتم استخدام زر "إيداع رصيد" واختيار (كاش أو تطبيق) ليسجل النظام "دفع مقدم".
4.  **تسجيل سحب نقدي**: في حال طلب الزبون مبلغاً نقدياً، يتم تسجيل المديونية وخصم المبلغ من جرد الكاش.
5.  **نهاية اليوم**: التوجه لشاشة الإحصائيات لمطابقة الكاش الفعلي وإغلاق اليومية.

## 🛠 التقنيات المستخدمة (Tech Stack)

*   **Framework:** Flutter (Windows, Android, Tablet).
*   **Database:** SQLite (v8 final schema) - Transaction-driven consistency.
*   **State Management:** Provider.
*   **Localization:** Full Arabic RTL support.

---

## 📂 هيكلية المشروع (Project Structure)

- `lib/models`: تعريف الكائنات (User, Invoice, FinancialTransaction, Purchase).
- `lib/services`: إدارة قاعدة البيانات `DatabaseService` ومنطق التحقق.
- `lib/screens`: واجهات المستخدم (Sales, Statistics, Customers, Settings, Calendar, Purchases).
